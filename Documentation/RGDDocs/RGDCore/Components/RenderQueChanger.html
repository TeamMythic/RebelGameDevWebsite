RenderQueChanger

The FPS Controller is a simple character capsule controller that has some unique options. You will find in the learning scene that there are many different parameters that you can alter for the state of the FPS controller.

What is a Render Que?

A render que is where the engine decides the render the object. Everything usually works great in unity until you work with translucent materials.

For instance, take a look at the 2.5D learning scene. If you move the player outside game view you will see the player is rendered above the transparent trigger.

How to fix this

To fix objects rendering above or behind transparent objects when they are not supposed to be. We can simply change the render que. Every material has one.

For ease of use, we have created a script that will change the render que upon objects spawning in. Jus simply add the RenderQueChanger component to the gameobject.

To see the changes in the editor. Click the 3 dots on the script components and select the function we made called Reseter.

Now you should see the object rendering correctly in the editor.

How To Reference in C#


                                   
    public class  TestScript : MonoBehaviour
    {
        //Get a reference to RenderQueChange component:
        [SerializeField] private  RenderQueChanger renderQueToChange;
        private   void  ChangeRenderQue(int newRenderQue)
        {
            //Call ResetRenderQue() to change the render que and update object's material:
            renderQueToChange.ResetRenderQue(newRenderQue);
        }
    }
