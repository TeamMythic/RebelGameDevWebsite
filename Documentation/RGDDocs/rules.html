Open Source Project Framework Rules

If you notice your github changes/additions are not added to the branch it's likely you did not follow the rules. Please feel free to contact us if you are having trouble.

Rule 1:

↦ No harassment or bullying. Any kind of harrassment will involve immediate whitelist from the project. You can appeal by messaging one of the Rebel Game Devs Officers.

Rule 2:

↦ Follow correct naming conventions for scripts.

Rule 3:

↦ You are not allowed to create layer masks in Unity since we are capped to 31 layers. If you are using layer masks for anything pleease use one of the already created layer masks. You are also only allowed 10 tags per user/team. The tag needs to follow the naming convention: YourFolderName_tagName.

Rule 4:

↦ You are to only upload to the dev branch. If this rule is not followed your push/merge to the branch will be denied.

Rule 5:

↦ Follow the "Asset File Storage Guidelines". You can find information about file storage inquires here. Not following this may result in your push/merge to the branch being denied.

Rule 6:

↦ Explicit or vulgar content is strictly prohibited and will result in an immediate whitelist from the project. You can appeal by messaging one of the Rebel Game Dev Officers.

Rule 7:

↦ We operate under a Open Source License, specifically the MIT open source license. We are giving you explicit freedom to do what you like with the content posted here. This also means anything you post on here loses all it's copyrights. We still want to maintain a structure of ownership. So please always comment in your scripts, scenes, etc. with your GitHub name. If you would like to check the license, please go to the repository as shown below.

Rule 8:

↦ You are prohibited from changing any folder other than the main JRPG folder or your team/personal folder. The Resources folder can be touched though, only if you make a folder with the exact name as your personal/team folder. If you would like to make a learning scene or utility component that is inside the RGD_Core folder please contact a officer through discord as you are not allowed to make changes to this folder without authorization. Even though you are only allowed to make changes to some folders you are still free to look and learn from other folders.
